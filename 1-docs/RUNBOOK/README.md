# Runbook 目录（索引）

> 规则：Runbook 按“问题域/现象类型”组织，而不是按“题目编号”组织。  
> 题目在 `labs/scenarios/`；Runbook 在这里沉淀“复用的处置路径”。

---

## 建议分组
- Web：Nginx 5xx、证书过期、上游超时
- systemd：服务起不来、重启风暴、依赖未就绪
- 容量：磁盘满、inode 满、日志暴涨
- 性能：CPU 高、load 高但 CPU 不高、内存压力/OOM
- 权限：应用读写失败、备份失败、sudo 误配
- 计划任务：cron 引发突刺、备份静默失败

---

## 新增 Runbook 的最低要求
- 现象（用户语言）与落位（资源/状态/队列/边界）
- 验证步骤（命令+预期）
- 最小修复与回滚
- 复发预防（监控/阈值/流程/SOP/容量）
